> **Instructions:** This is a template. Replace every section with your actual experience.
> The goal is to have 2-4 bullet variants per role so you can pick the most relevant ones for each application.
> Store the final version as `MASTER-BULLET-BANK.docx` using the `docx` skill.
> Run the `setup` skill to create this file from your existing resume, or build it manually below.

---

## How to Use This File

- **2-4 bullet options per role:** write a few different ways to describe the same work. For a data project, one bullet might emphasize the technical method; another might lead with the business outcome. During tailoring, Claude picks the one that best matches the job posting.
- **Quantify everything possible:** metrics, team sizes, budgets, percentages. Specific numbers beat vague adjectives.
- **Action verbs first:** Built, Designed, Engineered, Analyzed, Led, Managed, Reduced, Increased, Launched, etc.
- **No trailing periods on bullets.**
